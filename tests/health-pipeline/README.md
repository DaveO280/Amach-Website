# Health-pipeline data-integrity harness

Ends the "squeeze-the-balloon" stat fragility by giving the health pipeline a
fixed, verifiable reference to test back against. Full design: [`docs/architecture/13-data-integrity-harness.md`](../../docs/architecture/13-data-integrity-harness.md).

## Privacy: fixtures are NOT in git

Real Apple Health exports and the derived golden file are PII and live **outside
the repo**, referenced by `$AMACH_HEALTH_FIXTURE_DIR`. `.gitignore` blocks them
defensively (`HealthData*.csv`, `*.golden.json`, `/.health-fixtures/`, …). Only
code is versioned.

```
$AMACH_HEALTH_FIXTURE_DIR/
  export.csv         # frozen full-span Apple Health "Process Data" export
  stats.golden.json  # generated by generateGolden.ts, hand-verified vs Apple Health
```

## Running

```bash
# point at a local fixture dir (a gitignored .health-fixtures/ works)
export AMACH_HEALTH_FIXTURE_DIR="$PWD/.health-fixtures"
cp "/path/to/HealthData-export.csv" "$AMACH_HEALTH_FIXTURE_DIR/export.csv"

# (re)generate the golden file + print Apple-Health-confirmed sentinels
pnpm exec tsx tests/health-pipeline/reference/generateGolden.ts

# run the harness (skips cleanly if the env var is unset — never fails vacuously)
pnpm exec jest tests/health-pipeline
```

Without `AMACH_HEALTH_FIXTURE_DIR` the suites **skip** with an explanatory
message, so `pnpm test` stays green for contributors without the fixture. CI
injects the fixture from a secret store.

## What's here

| File                                 | Role                                                                                                                                                                                                 |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `reference/parseHealthCsv.ts`        | dependency-free CSV → typed records                                                                                                                                                                  |
| `reference/computeReferenceStats.ts` | **the truth**: raw records → daily/window/all-time stats, reusing the app's `METRIC_AGGREGATION_STRATEGIES`. Independent of the production pipeline by design. Policy choices documented at the top. |
| `reference/generateGolden.ts`        | regenerate `stats.golden.json` + print sentinels                                                                                                                                                     |
| `loadFixture.ts`                     | env-var fixture loader with clean skip                                                                                                                                                               |
| `__tests__/invariants.test.ts`       | Layer 2 — laws on any input (no metric/device dropped, daily=Σrecords, window-avg consistency)                                                                                                       |
| `__tests__/merge.test.ts`            | Layer 3 — appending newer days can't rewrite older days                                                                                                                                              |

## Confirmed sentinels (Apple-Health-verified)

Steps: 9/27/2025 = **25,712** (all-time high) · all-time avg **11,521** · raw low **28** · as-of 2026-06-16: 7d **11,690** / 30d **11,863** / 90d **11,006**. The reference engine reproduces all of these exactly.

## Not yet wired (next steps, per chapter 13 §Sequence)

- **Layer 1 golden test** asserting the _production_ daily-summary module converges to `stats.golden.json` — attaches once the four bucketing implementations are consolidated into one module (Phase 2).
- **Cross-surface consistency** (chart store vs. stat-card store) and **store-level merge idempotence** — need the production store wired in; that's the step that puts the stat-card divergence to bed.
- Sleep golden entries are `confidence: "provisional"` until the session-grouping policy is reconciled with `sleepDataProcessor`.
